# Motipe 프로젝트 회고

## 프로젝트 개요
축제 정보 AI 추천 웹 애플리케이션 | 5인 팀 | HTML, CSS, JS, Node.js, Express, Supabase, GitHub

---

## 진행 과정

처음에는 검색 기능을 담당했으나 AI 채팅 기능과 역할이 겹치는 부분이 있어 커뮤니티 게시글 DB 저장 기능으로 조율했습니다.
예상보다 빠르게 기능을 완성하여 담당자가 없던 지도 기능을 추가로 맡아 개발하게 되었습니다.
팀원들과의 소통이 원활하게 이루어졌고, 빠른 피드백을 주고받으며 프로젝트를 안정적으로 마무리할 수 있었습니다.

---

## 느낀 점

AI를 활용해 코드를 작성하고 분석하고 오류를 해결해 나가면서 얼마나 효율적으로 작업할 수 있는지 실감했습니다.
원하는 기능을 자연어로 풀어서 설명하고, 점점 원하는 방향으로 구체화되어 가는 과정이 흥미로웠습니다.
더 명확하게 의도를 전달하는 능력이 곧 AI를 잘 활용하는 역량이라는 것도 배울 수 있었습니다.
AI로 인해 개발자의 역할이 줄어드는 것이 아니라, 더 짧은 시간 안에 완성도 높은 결과물을 만들 수 있는 환경이 오고 있다고 느꼈습니다.

---

## 어려웠던 점

처음 사용해보는 Supabase에서 RLS 정책 개념이 낯설어 초반에 시행착오를 겪었습니다.
기능이 의도대로 동작하지 않을 때 오류 원인을 파악하는 데 시간이 걸렸지만, AI를 활용한 코드 분석으로 해결 방법을 찾아나갔습니다.

---

## 개선 사항

실질적인 개발 기간이 약 3일로 짧아 아쉬움이 남았습니다.
기간이 더 있었다면 AI 챗봇과 지도를 연동해 관련 축제 위치 확인 및 일정 연계 기능, 커뮤니티에서의 지도 활용 등을 구현했을 것입니다.

---

## 학습 내용

Git 브랜치 전략을 활용한 팀 협업 방식을 익혔습니다.
Supabase RLS 정책, Storage를 통한 이미지 파일 관리 등 새로운 기술들을 실제 프로젝트에 적용해볼 수 있었습니다.
AI 코드 리뷰를 통해 오류 원인 분석, 개발자 관점의 코드 개선 방향 등 코드를 바라보는 시각을 넓힐 수 있었습니다.
