## 1. switch 표현식 + while 무한루프 (Ex01)

```java
while (true) {
    int choice = sc.nextInt();
    String pokeName = switch (choice) {
        case 1 -> "이상해씨";
        case 2 -> "꼬부기";
        case 3 -> "파이리";
        case 4 -> "피카츄";
        default -> "";
    };
    if (pokeName.isBlank()) {
        System.out.println("다시 선택해주세요");
    } else {
        System.out.println("당신의 포켓몬은 " + pokeName + "입니다");
        break;
    }
}
```

- `switch` 표현식은 `->`를 쓰면 fall-through 없이 바로 값을 반환하고, 그 값을 변수에 즉시 대입할 수 있다.
- 잘못된 입력은 `default -> ""`로 빈 문자열을 주고 `isBlank()`로 걸러서 다시 입력받게 한 다음, 정상 입력이면 `break`로 `while(true)` 무한루프를 빠져나간다.

## 2. for / while 반복문, 중첩 for문 (Ex02)

정방향(`i++`)·역방향(`i--`) 반복을 for문과 while문 둘 다로 작성해봄.

```java
for (int i = 9; i > 0; i--) { ... }
// 역방향은 종료조건(i > 0)을 반드시 맞춰야 함, 안 그러면 무한루프
```

이중 for문(구구단), 삼중 for문도 작성해봄.

- **알게된 점**: 중첩 for문은 가장 안쪽 루프가 전체 횟수를 다 돌아야 그 한 칸 바깥 루프가 다음 단계로 넘어간다. 삼중 for문까지 가면 복잡도가 급격히 올라가서 실무·알고리즘에서는 잘 안 쓴다고 함.

## 3. 배열 선언 · 할당 · 다차원 배열 (Ex03, Ex04)

```java
int[] arr1 = new int[3];
arr1 = new int[5]; // 기존 배열은 못 늘리고, 새 배열로 통째로 교체
```

- 배열 인덱스는 0부터 시작, `arr1.length - 1`이 마지막 유효 인덱스. 범위를 넘기면 `ArrayIndexOutOfBoundsException`.
- 다차원 배열은 `int[][] arr4 = new int[2][3]`처럼 "바깥 칸 수 × 안쪽 칸 수"로 선언하고, `Arrays.deepToString()`으로 출력.
- Ex04에서는 배열을 거꾸로 순회(`arr.length - 1 - i`)하는 것과, 2차원 boolean 배열을 이중 for문으로 순회하는 것을 연습.

## 4. 향상된 for문(for-each)의 읽기 전용 특성 (Ex05)

```java
for (char c : charArr) {
    c -= 32; // 배열 안의 원본 값은 그대로, c는 복사본이라 바뀌지 않음
}
```

- for-each에서 꺼낸 변수는 원본의 복사본이라, 안에서 값을 바꿔도 원본 배열엔 반영되지 않는다.
- 다차원 배열을 for-each로 순회할 때는 `row`(한 행 배열) → `col`(그 행의 원소) 순으로 중첩해서 순회하고, 이때는 인덱스를 쓸 수 없다는 게 일반 for문과 다른 점.

## 5. 컬렉션 - List / ArrayList (Ex06)

```java
List<String> list3 = new ArrayList<>(); // 인터페이스 타입으로 선언, <> 생략 가능
List<Integer> list4 = List.of(1, 2, 3, 5, 8, 13);
List<String> list5 = Arrays.asList(stringArr);
list5.set(2, "C");
```

- `ArrayList<Integer>`처럼 제네릭은 참조 타입만 가능해서 `int` 대신 `Integer`를 써야 함.
- `add`(추가) / `get`,`set`(인덱스 기반 조회·수정) / `remove`(삭제) / `size()`로 길이 확인.
- for문과 for-each 둘 다로 순회 연습.

## 6. 컬렉션 - Map / HashMap (Ex07)

```java
map.put("banana", map.get("banana") * 10);
map.put("cherry", map.getOrDefault("cherry", 0) * 10); // null 방지
```

- `put`(추가/수정), `get`(키로 조회), `remove`(키로 삭제), `containsKey`/`containsValue`, `keySet`/`values`/`entrySet`으로 순회.

## 7. 컬렉션 - Set / HashSet (Ex08)

```java
Set<String> set2 = new HashSet<>(List.of("감", "밤", "대추", "대추"));
```

- 같은 값을 여러 번 `add`해도 한 번만 저장됨 (중복 제거).
- `add`/`remove`/`contains`로 조작, 순서는 보장되지 않음.

## 오늘 헷갈렸던 부분 — 문제 → 시도 → 해결 → 알게된 점

### (1) `Arrays.asList()`로 만든 리스트, set은 되는데 add는 될까?

**문제**: Ex06에서 `Arrays.asList(stringArr)`로 만든 `list5`에 `set()`이 잘 동작해서, 이게 일반 `ArrayList`랑 완전히 똑같이 쓸 수 있는 건지 궁금했다.

**시도**: `list5.set(2, "C")`와 for문 안에서 `list5.set(i, list5.get(i).repeat(2))`로 값을 두 배로 늘리는 걸 직접 실행해서 동작을 확인.

**해결**: `Arrays.asList()`는 원본 배열을 그대로 감싸는 **고정 크기 리스트**라서, 인덱스로 값을 바꾸는 `set()`은 가능하지만 크기를 바꾸는 `add()`/`remove()`는 지원하지 않는다는 걸 정리했다.

**알게된 점**: 같은 `List` 타입으로 보여도 내부 구현(`ArrayList`인지 `Arrays.asList`의 결과인지)에 따라 실제로 쓸 수 있는 메서드가 달라진다. `add()`를 호출했다면 `UnsupportedOperationException`이 났을 것.

### (2) `map.get()`이 null을 줄 때 생기는 NullPointerException

**문제**: Ex07에서 없는 키(`"cherry"`)를 `map.get()`으로 조회한 다음 바로 `* 10` 연산을 하면 NPE가 난다는 주석을 발견했다.

**시도**: 주석으로 막혀 있던 코드를 보면서 왜 터지는지 따라가봄 — `get()`이 `null`을 반환하고, `Integer`를 `int`로 자동 변환(`intValue()`)하려는 시점에서 그 대상이 `null`이라 터지는 구조였다.

**해결**: `map.getOrDefault("cherry", 0) * 10`처럼 키가 없을 때 쓸 기본값을 지정해서 null 자체를 피하는 방식으로 해결.

**알게된 점**: `get()`은 키가 없으면 `null`을 돌려주므로, 받은 값을 곧바로 연산에 쓸 가능성이 있다면 `getOrDefault`를 기본으로 쓰는 습관을 들이는 게 안전하다.

### (3) 배열은 고정 크기인데, 재할당은 왜 되지?

**문제**: Ex03에서 `arr1 = new int[3]`으로 만든 배열을 다시 `arr1 = new int[5]`로 바꾸는 코드가 있어서, "배열은 고정 크기"라는 설명과 모순되는 것처럼 보여 헷갈렸다.

**시도**: 재할당 전후로 `arr1`을 출력해보면서 실제로 어떤 게 바뀌는지 확인.

**해결**: 배열 **인스턴스 자체**의 길이는 못 바꾸지만, `arr1`이라는 **참조 변수**에 완전히 새로운 배열을 다시 대입하는 건 가능하다는 걸 확인했다. 즉 "이미 만들어진 배열"은 불변이고, "변수가 가리키는 대상"은 다른 객체 변수와 똑같이 바꿀 수 있는 것.

**알게된 점**: 배열의 불변성은 인스턴스 단위의 제약이고, 참조 변수 자체의 재할당 가능 여부와는 별개의 이야기라는 걸 명확히 구분하게 됐다.
